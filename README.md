# ПТО «Север» — Контроль СМР (082.024)

Веб-приложение ПТО для объекта **«Спорткомплекс Север», г. Северодвинск.
Плавательный бассейн (шифр 082.024)**: ОЖР, ГПР, табель с календарём присутствия,
ТМЦ, планы задач бригад, расчёт бетона, армирование, АОСР, контроль прочности,
чек-листы, инструктажи, наряды-допуски, анализ проекта по чертежам, нормативы,
SCAD AI САПР, дашборд.

---

## 🔐 Безопасность — прочитать первым делом

Репозиторий **публичный**, поэтому никаких секретов в коде нет. Всё хранится
**в переменных окружения Render** (Environment Variables):

| Переменная | Назначение |
|-----------|-----------|
| `ZVENO_KEY` | ключ ZVENO (ИИ). В коде его нет — подставляет сервер. |
| `APP_LOGIN` | логин для входа в приложение |
| `APP_PASSWORD` | пароль для входа |
| `AUTH_SECRET` | любая длинная случайная строка (подпись сессии), напр. 30+ символов |

**Вход в приложение:** при открытии сайта показывается страница логина.
Логин/пароль сверяются **на сервере** с `APP_LOGIN`/`APP_PASSWORD`. После входа
выдаётся сессия на 12 часов (HttpOnly-cookie). Без входа ни приложение, ни ИИ
(`/api/chat`) не отдаются.

✅ Логин, пароль и ключ ZVENO никогда не попадают в браузер и в репозиторий.
❌ Не вписывайте их в `PTO_SEVER_APP.html` или `server.js`.

> Старый ключ ZVENO (был в коде раньше) считайте скомпрометированным —
> **отзовите его** и выпустите новый.

## 🚀 Размещение на GitHub + Render — шаг за шагом

### Шаг 1. Залить код на GitHub

**Через сайт (проще):**
1. github.com → **«+» → New repository**.
2. Имя `pto-sever`, можно **Public** (ключа в коде нет — это безопасно).
3. **Create repository → Add file → Upload files** → перетащите:
   `PTO_SEVER_APP.html`, `server.js`, `package.json`, `README.md`, `.gitignore`.
4. **Commit changes**.

### Шаг 2. Развернуть на Render

1. Зайдите на **https://render.com** → войдите через GitHub.
2. **New + → Web Service**.
3. Выберите репозиторий `pto-sever` → **Connect**.
4. Настройки:
   - **Name**: `pto-sever` (часть будущего адреса)
   - **Runtime**: `Node`
   - **Build Command**: оставьте пустым (зависимостей нет)
   - **Start Command**: `node server.js`
   - **Instance Type**: `Free`
5. Раскройте **Advanced → Add Environment Variable** и добавьте:
   - `ZVENO_KEY` = ваш ключ ZVENO (`sk-...`)
   - `APP_LOGIN` = придуманный логин (напр. `pto`)
   - `APP_PASSWORD` = придуманный пароль (надёжный)
   - `AUTH_SECRET` = длинная случайная строка (напр. `k7Hx9...` 30+ символов)
   - (необязательно) `ZVENO_URL` = `https://api.zveno.ai/v1/chat/completions`
6. **Create Web Service**. Render соберёт и запустит (1–3 минуты).
7. Готово. Адрес вида **`https://pto-sever.onrender.com`** — это приложение.
   ИИ работает через `/api/chat`, ключ — на сервере.

### Шаг 3. Проверить

- Откройте `https://pto-sever.onrender.com` — появится **страница входа**.
- Введите `APP_LOGIN` / `APP_PASSWORD` → откроется приложение.
- **Данные / Настройки**: режим «ZVENO AI (через прокси)», поле ключа **пустое**,
  endpoint **`/api/chat`** (так и оставьте).
- Проверьте любую ИИ-функцию (SCAD AI САПР, «развернуть запись ОЖР»).

> ⚠️ На бесплатном плане Render сервис «засыпает» после 15 минут простоя,
> первый запрос после паузы идёт ~30–50 секунд (просыпается). Это нормально.

## ❗ Честные ограничения

- Поиск по чертежу — по **текстовому слою** PDF. Для «картиночных» листов без текста
  есть **OCR** (кнопка «Распознать текст» в Анализе проекта): читает надписи/марки
  с изображения и добавляет в поиск. Медленно (10–40 сек/лист), геометрию не распознаёт.
- Объёмы — из спецификаций проекта; геометрию чертежа программа не «видит».
- Нормы выработки/набор прочности — ориентировочные (сметы — ЕНиР/ГЭСН, лаборатория).
- Полные тексты СП/ГОСТ не встроены (авторское право) — указатели и ссылки.
