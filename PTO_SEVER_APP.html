1. Начало работы и сохранение базы

Запускайте через start.bat (поднимает локальный сервер, открывает в браузере) — тогда работает автосохранение в файл. При запуске появится окно выбора: открыть файл базы, загрузить TXT или продолжить с последней работы.

В разделе «Данные / Настройки» нажмите «Создать файл БД» (Chrome/Edge через сервер) — после этого все изменения автоматически пишутся в PTO_DATA_SEVER.txt. Резервно работает «Сохранить БД (TXT)».

Пример: утром открыли start.bat → в окне «Открыть файл базы» выбрали PTO_DATA_SEVER.txt → работаете весь день, всё пишется само → вечером кнопка «✕ Закрыть» предложит дозаписать.

2. Общий журнал работ (ОЖР)

Три способа внести запись: (а) выбрать вид работ и элемент из списков → «Сформировать из списка»; (б) написать черновик своими словами → «Развернуть по проекту»; (в) импорт из Excel.

Заполните погоду (кнопка «🌤 Загрузить погоду» подтянет с интернета), объём и единицу. При t<5°C по бетону программа предупредит о зимнем бетонировании.

Пример заполнения: Дата 21.06.2026 · Захватка «Ростверки по оси К, захватка 1» · Вид работ «Бетонирование» · Элемент «РМ-1.3» → текст сформируется автоматически · Объём «1.5» · Ед. «м³» · Погода «+12°C, ясно».

3. График работ (ГПР) и расчёт сроков

Раздел «Расчёт сроков (выработка)»: вид работ → норма подставится → укажите объём, число рабочих, смены → «Рассчитать срок» → «Добавить в график». В ГПР увидите диаграмму Ганта с план/факт (факт берётся из объёмов ОЖР) и колонку «% выполнения».

Пример: «Вязка арматурного каркаса», объём 0.9 т, норма 0.12 т/чел.-смену, 4 рабочих → 0.9÷(0.12×4)=2 смены → даты подставятся, позиция уйдёт в ГПР.

4. Сотрудники, табель, КТУ

Сначала добавьте сотрудников (ФИО, должность, тариф). В табеле отмечайте явку, часы считаются автоматически (минус обед). КТУ — коэффициент трудового участия; нажмите «ⓘ как назначать?» для подробной шкалы.

Пример КТУ: перевыполнил норму, помогал бригаде → КТУ 1,2; брак с переделкой → КТУ 0,7.

5. Учёт ТМЦ

Операции прихода/расхода/возврата материалов. Остаток по каждому считается сам, отрицательные подсвечиваются красным.

Пример: Приход «Арматура А500С ∅16», 2.5 т, поставщик ООО, ТН №123 → затем Расход 1.5 т «на ростверк РМ-1.3» → остаток 1.0 т.

6. Планы задач бригад + лист чертежа

Создайте план: дата, бригада, захватка, выберите лист из загруженного PDF. У плана добавляйте задачи с отметкой выполнения и комментариями. Кнопка «🖍 Лист» открывает выбранный лист — маркером обведите участки работ. «→ОЖР» вносит задачу в журнал, «→ГПР» — в график.

Пример: План «Бригада №2 · 21.06 · оси К-Л», лист «КЖ0 л.34» → обвели зону ростверков → задача «связать каркас РМ-1.3, 5 шт» → выполнено → «→ОЖР».

7. Анализ проекта и чертежи

Поиск по 33 проверенным элементам базы (РМ, Кбм, бетоны, арматура) + по загруженным PDF. Загрузите PDF чертежей — извлечённые марки добавятся в списки, клик по результату откроет нужный лист. «🔍 Найти на чертеже» ищет по осям/марке.

Пример: ввели «РМ-1.3» → карточка с армированием и ссылкой «КЖ0 л.17» → клик открывает лист (если PDF загружен).

8. Расчёт бетона и армирование

Расчёт бетона: укажите оси, выберите элементы и количество → объём посчитается из спецификаций (+2% потери). Карта заливки — обведите области на листе. Армирование: поиск по проекту/правилам, глубокий анализ PDF (диаметры/классы), вырезка узлов с чертежа, решение спорных вопросов.

Пример: оси «4-5», Кбм-5 × 3 шт → 0.518×3=1.554 м³ → с потерями ~1.59 м³ → «Внести в ОЖР».

9. Контроль: АОСР, прочность бетона, чек-листы

АОСР — акты скрытых работ, можно создать из записи ОЖР. Контроль прочности — журнал кубиков, набор прочности по дням. Чек-листы — готовые шаблоны приёмки (бетонирование/арматура/опалубка) с отметками.

Пример: перед заливкой создали чек-лист «Готовность к бетонированию» → отметили все 10 пунктов → 100% «готово» → распечатали.

10. Охрана труда: инструктажи, наряды-допуски

Журнал инструктажей (ГОСТ 12.0.004): вид, сотрудник, инструктирующий. Наряд-допуск на опасные работы: вид, место, меры безопасности, срок, бригада. Оба печатаются по форме.

Пример наряда: «Работы в котловане», место «оси К, отм. −3.000», меры «крепление откосов, СИЗ, газоанализ», срок 21–22.06, бригада 4 чел.

11. Таблицы (импорт и экспорт XLS)

Журналы (ОЖР, табель, ТМЦ, ГПР, контроль бетона) можно выгрузить в Excel кнопкой «⬇ Excel» и загрузить из Excel. Для ОЖР есть шаблон Excel с правильными заголовками — скачайте, заполните, импортируйте.

Пример: «Скачать шаблон Excel» → заполнили 20 строк работ в Excel → «Импорт из Excel» → программа создала 20 записей ОЖР.

12. Честные ограничения программы (важно)

Чтобы не было ложных ожиданий:

  • Программа не распознаёт геометрию чертежа по пикселям — объёмы берутся из спецификаций, области отмечаются вручную маркером.
  • Поиск по PDF идёт по текстовому слою — на «графических» чертежах без текста может не находить.
  • Нормы выработки и набор прочности — ориентировочные; для смет и приёмки используйте ЕНиР/ГЭСН и лабораторные испытания.
  • Погода — текущая (Open-Meteo), не исторический архив.
  • Автозапись в файл — только Chrome/Edge через локальный сервер (start.bat), не file://.
  • Полные тексты норм не встроены (авторское право) — даны указатели и ссылки на официальные источники.
  • ИИ-функции (ZVENO) работают при включённом ключе; иначе — локальные шаблоны.

Дополнительно: загрузка PDF чертежей (опционально)

Честно о возможностях: браузер извлекает только текстовый слой PDF. На «тяжёлых» чертежах САПР текст бывает рваным или отсутствует (графика) — тогда полнотекстовый поиск по загруженному PDF может не находить марки. Встроенная база выше работает всегда. PDF-просмотр (с зумом) работает для визуального контроля листа.

🔎 OCR — распознать текст на «картиночном» листе

Когда нужно: если у листа нет текстового слоя (чистая графика/скан) и поиск его не находит. OCR прочитает надписи и марки с изображения и добавит их в поиск. Не распознаёт геометрию чертежа. Медленно: 10–40 сек на лист. Нужен интернет.
⚠ Пакетный OCR обрабатывает все листы без текстового слоя во всех загруженных PDF. Это долго: ~10–40 сек на лист (тома в сотни листов — десятки минут). Можно прервать. Результаты сохраняются в базу.
После загрузки PDF извлечённые марки автоматически добавляются в выпадающий список «Выбрать элемент из списка проекта» (с пометкой «из PDF»).

Анализ выполнения работ и прогноз сроков

Анализ строится по записям ОЖР с объёмами по датам. Важно про единицы: прогноз считается отдельно по каждой единице измерения (м³ бетона, т арматуры, шт свай и т.д.) — разные единицы не суммируются. Выберите единицу, и программа возьмёт только записи ОЖР с этой единицей, посчитает темп (ед./день) и спрогнозирует дату завершения по плановому объёму в той же единице.

Выберите вид работ и укажите плановый объём, затем нажмите «Рассчитать прогноз».

График производства работ (ГПР)

Календарный график: список работ с датами начала/окончания, объёмами и ответственными. Внизу — диаграмма Ганта. Можно сформировать строки из перечня работ проекта.
Расчёт сроков по выработке: укажите вид работы, объём и число рабочих — программа посчитает дни и подставит дату окончания (по справочнику норм выработки на человека). ⓘ справочник выработки

Создание ППР с помощью SCAD AI САПР

SCAD AI САПР (ZVENO/совместимый) формирует черновик проекта производства работ на основе встроенной базы проекта и норм. Это черновик-заготовка для инженера ПТО, требует проверки и доработки.
Честно: без включённого API (вкладка «Настройки») работает локальный шаблонный генератор — даёт структуру ППР по выбранной работе и данным проекта. С API ответ формирует нейросеть по тому же контексту. Готовый ППР всё равно проверяет и утверждает инженер.

Выберите вид работ и нажмите «Сформировать черновик ППР».

Отвечает по встроенной базе проекта + (если включён) ZVENO AI. Запрещены выдумки: если данных нет — так и скажет. Ссылается на листы проекта.

Приоритет — данные чертежа. Если на чертеже нет ответа, ИИ привлекает строительные нормативы (армирование, бетонирование, разбивка) с указанием документа, пункта и ссылкой на первоисточник. Важно: номера пунктов/страниц от ИИ нужно проверять по официальному тексту (docs.cntd.ru) — модель может ошибаться в нумерации. Рекомендуется модель Claude.

Анализ проекта (ИИ) — отчёт об ошибках

Честно о возможностях: ИИ анализирует текст загруженных PDF + базу проекта. Он может пропустить ошибки или принять верное за ошибку — это помощник для проверки, а не замена экспертизы. Справочник: как назначать КТУ

КТУ (коэффициент трудового участия) — множитель, которым корректируют оплату/распределение приработка с учётом реального вклада работника: выработки, качества, сложности, дисциплины. В программе КТУ умножается на (часы × тариф).

Базовый принцип

Норма (средний вклад)КТУ = 1,0
Повышают за: перевыполнение, высокое качество, сложность, наставничество, совмещение> 1,0
Понижают за: брак, простои по вине работника, нарушения ТБ/дисциплины, низкую выработку< 1,0

Ориентировочная шкала

КТУКогда назначать
1,3–1,5Выдающийся вклад: значительное перевыполнение, сложный/ответственный участок, безупречное качество, замена/наставничество
1,1–1,25Выше среднего: перевыполнение нормы, инициатива, совмещение операций, помощь бригаде
1,0Норма: задание выполнено в срок и с надлежащим качеством, без замечаний
0,8–0,95Ниже среднего: недовыполнение нормы, мелкие замечания по качеству, опоздания
0,5–0,75Существенные нарушения: брак с переделкой, простой по вине работника, нарушение ТБ
0Прогул, грубое нарушение, отказ от работы (за этот день участие не засчитывается)

Факторы повышения (+)

  • перевыполнение сменного задания / высокая выработка;
  • качество выше требований, отсутствие переделок;
  • сложность и ответственность участка (узлы, стеснённые условия);
  • совмещение профессий/операций, взаимозаменяемость;
  • наставничество, помощь менее опытным;
  • инициатива, рацпредложения, бережное отношение к технике/материалам.

Факторы понижения (−)

  • невыполнение нормы выработки без уважительной причины;
  • брак, дефекты, переделки по вине работника;
  • нарушение технологии, требований ППР/чертежей;
  • нарушение трудовой дисциплины (опоздания, ранний уход, прогул);
  • нарушение правил охраны труда и пожарной безопасности;
  • простои и порча материала/инструмента по вине работника.

Как назначать на практике

  1. Базово всем — КТУ 1,0.
  2. Бригадир/мастер по итогам смены (недели) корректирует вверх/вниз по факторам выше.
  3. Решение фиксируется (протокол бригады / решение мастера), доводится до работника.
  4. КТУ применяют к распределяемой части (приработок/премия), как правило не к гарантированной тарифной части — порядок устанавливает положение об оплате труда организации.
  5. Спорные случаи рассматриваются с участием бригады/профкома.
Важно: КТУ и порядок его применения устанавливаются локальными актами организации (положение об оплате труда, коллективный договор) в рамках ТК РФ. Эта шкала — методический ориентир. КТУ не должен снижать оплату ниже гарантий, установленных трудовым договором и законом.
