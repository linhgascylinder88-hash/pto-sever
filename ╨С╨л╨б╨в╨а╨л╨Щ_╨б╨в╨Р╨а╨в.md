# 🚀 Шпаргалка: запуск за 5 шагов

Краткая версия. Подробности — в `README.md`.

## 1. GitHub
- github.com → New repository → имя `pto-sever` → **Public** (секретов в коде нет) → Create
- Add file → Upload files → перетащите ВСЕ файлы из архива → Commit

## 2. Render
- render.com → войти через GitHub → **New + → Web Service** → выбрать репозиторий `pto-sever`
- Runtime: **Node** · Build Command: *(пусто)* · Start Command: **`node server.js`** · план **Free**

## 3. Переменные окружения (Advanced → Environment Variable)
Добавьте **4 переменные**:

| Key | Value |
|-----|-------|
| `ZVENO_KEY` | ваш ключ ZVENO (`sk-...`) |
| `APP_LOGIN` | логин для входа (придумайте) |
| `APP_PASSWORD` | пароль (придумайте надёжный) |
| `AUTH_SECRET` | длинная случайная строка 30+ символов |

→ **Create Web Service**

## 4. Готово
Через 1–3 минуты — адрес вида `https://pto-sever.onrender.com`.
Откроется **страница входа** → введите `APP_LOGIN` / `APP_PASSWORD` → приложение.

## 5. Первая настройка в приложении
- **Данные / Настройки**: режим «ZVENO AI (через прокси)», ключ оставьте **пустым**, endpoint **`/api/chat`**
- **Реквизиты / Ответственные**: впишите данные организации (для печатных форм)
- Можно начинать работу

---

## ⚠️ Не забыть
- Старый ключ ZVENO (был в коде раньше) — **отзовите** в кабинете ZVENO, выпустите новый.
- Бесплатный Render «засыпает» после 15 мин простоя — первый запрос после паузы ~30-50 сек.
- Рабочая база `PTO_DATA_SEVER.txt` хранится у вас (в репозиторий не попадает — так и нужно).

## 🔄 Обновление
Залили новую версию файла в GitHub → Render пересоберёт сам (auto-deploy).
